为什么必须有后端代理

静态站前端代码会被所有访客看到，所以这里把接口治理放到 Netlify Functions：前端只调用本站后端，后端控制可访问的 API 白名单、请求头、超时、缓存和未来密钥读取，避免把账号额度和密钥暴露出去。

后端代理标准

  • 前端只能传白名单 id，不能传任意外部 URL。
  • 后端统一处理超时、错误和缓存响应头。
  • 失败结果返回 statusCode、errorType、possibleCauses、suggestions 和 latencyMs。
  • 按访客限流；全量健康检查仅允许携带服务端环境变量令牌的任务调用。
  • 能服务博客、论文、项目、材料或设计展示。
