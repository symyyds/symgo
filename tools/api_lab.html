INTEGRATION STRATEGY

public-apis 接入策略

这个页面把 public-apis 中和个人博客最相关的能力拆成三类：可以在 Netlify 静态前端直接请求的实时 API、适合沉淀到论文/项目/博客/求职页面的数据源、以及需要密钥时必须走后端代理的候选 API。

为什么不把 API Key 写进网页

静态站前端代码会被所有访客看到。需要申请 Key 的接口可以后续迁移到 Netlify Functions 或其他后端代理；当前页面只默认启用无需密钥的公开 API，避免把账号额度和密钥暴露出去。

已筛选标准

  • 无需 API Key 或 OAuth。
  • HTTPS 可访问，适合 Netlify 静态站。
  • 能服务博客、论文、项目、材料或设计展示。
BACKEND CANDIDATES

适合后续走 Netlify Functions 的 API

这些 API 和个人站也相关，但需要 Key、OAuth、服务端签名或更严格的配额治理。正确做法是把密钥放在 Netlify 环境变量里，由 Functions 代理请求。
