问题场景

电力物联网终端存在厂家多、型号多、协议不统一、点表参数依赖人工配置、安全认证链条不完整等问题，影响规模化接入和统一管控。

端边底座

端侧采用 BearPi-HM Nano 与 OpenHarmony，边缘侧采用 OrangePi KunPeng Pro 与 openEuler，并用 Docker 部署 EdgeX Foundry 微服务框架。

可信管控

方案覆盖 SM2 签名验签、加密传输、nonce 防重放、安全接入 SDK、证书链路和平台侧可信连接，同时扩展射频指纹、协议解析和漏洞协同探测研究。

02

技术路线与系统链路

项目以国产信创端边环境为底座，用 EdgeX Foundry 统一管理设备元信息、设备服务和命令通道，降低后续设备接入的重复配置成本。

  • 边缘侧通过 Docker Compose 启动 core-data、core-metadata、core-command、app-service、MQTT Broker 以及 MQTT/Modbus 设备服务。
  • MQTT 设备通过注册主题上报 deviceId、deviceType、protocol、公钥等信息，边缘侧解析后完成设备创建与 Device Profile 绑定。
  • Modbus 设备通过寄存器区段承载注册信息，边缘设备服务读取后生成设备元数据和周期采集任务。
  • Device Profile 将不同协议设备资源抽象为统一的 Device Resource，让平台侧读取数据时不直接感知底层协议差异。
03

示范应用与联调验证

示范应用围绕配电站房环境监测、供电所微电网设备接入等场景展开，验证端设备、边缘代理、物联管理平台之间的数据链路。

  • 智能开关通过 Modbus 完成自动注册，采集电压、电流、功率和开关状态。
  • 烟感传感器通过 MQTT 完成自动注册，周期性上报烟感 PPM 数据。
  • 边缘代理设备经安全接入 SDK 和证书链路与物联管理平台对接，平台侧支持设备列表查询、历史数据检索和命令下发。
  • 端设备写入网络配置后可在平台自动注册展示，获取数据后可实时向物联平台上报。
04

可信认证与智能扩展

可信管控部分不是只做单点加密，而是把身份表示、通信保护、边缘侧流量控制和主动防护研究合在一起。

  • 端边通信集成 SM2 国密算法，用于签名验签、加密传输和 nonce 防重放保护。
  • 查新材料中归纳了“物理层射频指纹 + 协议层国密算法 + 系统层多智能体协同漏洞探测”的三维可信管控思路。
  • CNN-LSTM 设备识别与边缘智能响应用于增强异构终端识别、行为理解和数据管控能力。
  • 协议智能解析、数字孪生沙箱和多智能体强化学习漏洞探测作为后续安全扩展方向沉淀到成果材料中。
